# 02 微信聊天记录先看什么

建议时长：60-90 秒

## 逐字稿

拿到一份微信聊天记录，第一步先看结构。

你需要把它拆成五个字段：谁发的，什么时候发的，发的是什么类型，具体内容是什么，消息编号是什么。

这五个字段对应五类关系信号。

“谁发的”，看主动性和参与度。

“什么时候发”，看回复速度、等待时间和沉默长度。

“消息类型”，看这段关系更依赖文字、语音、图片，还是表情。

“具体内容”，看情绪、支持、请求、解释、道歉、玩笑、冲突和亲密表达。

“消息编号”，用来保证顺序稳定，方便回看某一段互动。

接下来，把聊天按时间切成一段一段。比如，30 分钟以内连续互动，可以算一个小会话；隔了几个小时再聊，就是另一个会话。

关系分析最怕把所有消息混在一起看。早上吵架、晚上和好，放在一起平均，就会把关键变化抹平。

所以，微信聊天记录的第一步，是建一张干净的 message table。

每一行是一条消息，每一列是一个可分析字段。

有了这张表，你才能看出谁在推动关系，谁在回应关系，关系在什么时候升温，什么时候变冷，什么时候开始修复。

## 拍摄节奏

- 0-8 秒：讲“先看结构”。
- 8-35 秒：逐个解释五个字段。
- 35-60 秒：讲时间切段。
- 60-85 秒：讲 message table 的用途。

## 画面提示

- 展示一个脱敏表格：sender、timestamp、type、content、platformMessageId。
- 用色块标出一天内的不同会话段。
- 展示“消息流 -> message table -> 关系特征”的流程图。
